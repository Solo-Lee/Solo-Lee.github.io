用Hexo搭建属于自己的博客(一)

用Hexo搭建一个静态博客并在本地展示

Chat 闲聊

  一直想写一个教学系列的文章,于是就开了这个Hexo的坑.在 满足自身的愿望同时呢,也是当做自己学习上的一个笔记吧.
  网上关于Hexo的教程也有很多了,写这个教学的文章,我的目标是能让一个没有编程经验的人也能够搭建出自己的博客,上传自己的文章.
  希望大家阅读我的文章能有所收获…最后在搭建的过程中有遇见什么问题可以问我,我会尽量为你解答.
下面开始进入我们的正题.

Hexo简介

简单的说Hexo是一个基于 Node.js 的静态博客框架.Hexo有很好的易用性.对于各类人群都很友好.

Achieve 实现

1. 环境的准备

在搭建自己的博客前我们需要做好以下准备:

  • 注册一个Github帐号
  • 安装 Git
  • 安装 Node.js

  我用的是Winows的系统,所以教程中所有的操作都是在Windows上进行的.
在安装过程中只需要一直下一步就好了.

2. Hexo的安装

  如果你已经安装上述环境的准备,那么接下来我们就可以开始Hexo的安装了.
使用组合快捷键Win+R打开运行对话框,输入cmd回车打开命令提示符.
我们在新打开的命令提示符里依次输入以下指令:

npm install -g hexo-cli 安装Hexo

npm install 安装相关的依赖包

3. 博客的初始化

  选择一个磁盘新建一个文件夹．这里我在Ｅ盘根目录下新建了一个名为blog的文件夹．
可以通过指令在命令提示符里创建，也可以自己手动在磁盘里创建，选择自己熟悉的方式即可．

mkdir E:\blog
创建成功后我们跳转到创建的目录上或者打开文件夹对空白处单击右键选择Git Bash Here：

cd /d E:\blog
在命令提示符中或Git的shell工具中输入以下指令:

hexo init 初始化Hexo
执行成功后我们可以看见原本空的文件夹里多出了很多的文件．

4. 本地服务的启动

在启动本地服务器时我们需要生成静态文件,在命令提示符或shell工具中输入指令：

hexo generate 也可以输入hexo g．
执行成功后我们可以看到blog文件夹下多出了一个名为public的文件夹，这样我们博客的静态文件就生成成功了．
我们再在命令提示符或shell工具中输入指令：

hexo server 也可以输入hexo s．

5. 本地查看博客

最后到了最激动人心的时候了,做了那么多的工作,终于可以查看自己制作的博客了．
打开浏览器,在地址栏输入localhost:4000回车.
当自己搭建的博客展示在面前,是不是特别有成就感呢.
